Metodología — Sesión 1

Diagramas en sintaxis PlantUML. El navegador pide el render a Kroki (requiere internet). Editá el código en cada bloque pre.plantuml-source y recargá la página.

Diagrama de clases

            @startuml
left to right direction

class Usuario {
  - id : UUID
  - email : String
  + registrarse()
  + iniciarSesion()
}

class Pedido {
  - id : UUID
  - fecha : Date
  - estado : EstadoPedido
  + calcularTotal() : Money
}

class LineaPedido {
  - cantidad : int
  - precioUnitario : Money
}

Usuario "1" --> "*" Pedido : realiza
Pedido "1" *-- "*" LineaPedido : contiene
@enduml

Diagrama de secuencia

@startuml
actor Usuario
participant "Web" as W
participant "API" as API
participant "Base de datos" as DB

Usuario -> W : Solicita checkout
W -> API : POST /pedidos
API -> DB : INSERT pedido
DB --> API : id pedido
API --> W : 201 Created
W --> Usuario : Confirmación
@enduml

Diagrama de estados

@startuml
[*] --> Borrador
Borrador --> Enviado : enviar
Enviado --> EnRevision : recibir
EnRevision --> Aprobado : aprobar
EnRevision --> Rechazado : rechazar
Aprobado --> [*]
Rechazado --> Borrador : corregir
@enduml
Sin conexión o si Kroki no responde, los diagramas no se verán; el texto PlantUML sigue en la página para copiar/pegar en un editor local con PlantUML.
